# 개발 가이드 문서 체계

이 폴더는 기능 개발 시 작성하는 **3단계 문서 체계**의 템플릿과 규칙을 정의한다.

## 문서 계층

```
요구사항 → 설계 → 구현
  prd.md  → plan.md → task.md
```

각 단계는 이전 단계의 산출물을 기반으로 작성한다.  
이전 단계 문서가 확정되지 않으면 다음 단계로 넘어가지 않는다.

## 파일별 역할 요약

| 파일 | 작성 주체 | 핵심 질문 | 기술 상세도 |
|------|----------|----------|------------|
| `prd.md` | 기획/도메인 | **무엇을** 만드는가 | 없음 |
| `plan.md` | 아키텍처/설계 | **어떻게** 구조화하는가 | 중간 |
| `task.md` | 개발자 | **어떤 코드를** 작성하는가 | 높음 |

## 작업 흐름

1. `prd.md` 작성 → 요구사항 합의
2. `plan.md` 작성 → 설계 검토
3. `task.md` 작성 → 구현 착수
4. 구현 완료 후 각 문서의 상태(Status)를 업데이트

## 파일 위치

기능 단위로 하위 폴더를 만들어 관리한다.

```
.guide/
├── README.md          # 이 파일
├── prd-guide.md       # PRD 작성 규칙 및 템플릿
├── plan-guide.md      # Plan 작성 규칙 및 템플릿
└── task-guide.md      # Task 작성 규칙 및 템플릿
```

실제 작업 문서는 프로젝트 루트나 기능 폴더에 위치시킨다.

```
{feature}/
├── prd.md
├── plan.md
└── task.md
```
