# PRD: DART 대량보유보고서 수집 파이프라인

## 상태
- 작성일: 2026-04-23
- 최종 수정: 2026-04-23
- 상태: Confirmed

---

## 배경

현재 서비스는 공공데이터포털에서 국민연금의 포트폴리오 데이터를 수집한다.
이 데이터는 매년 12월 31일 기준으로 작성되며 이듬해 9월경에 공개된다.
즉, 사용자는 최대 **9개월 지연된** 데이터만 확인할 수 있다.

금융감독원 DART(전자공시시스템)에는 국민연금이 특정 종목을 **5% 이상 보유하다가
1% 이상 변동**이 발생할 때마다 의무적으로 제출하는 대량보유보고서가 공개된다.
이 보고서는 변동 발생 후 **5 영업일 이내** 공시되므로,
대형주 중심의 매매 변동을 공공데이터포털 대비 훨씬 빠르게 파악할 수 있다.

---

## 목표

- 국민연금이 5% 이상 보유한 종목의 **지분 변동을 수일 내에 확인**할 수 있게 한다.
- 기존 연간 데이터와 DART 데이터를 **통합하여 하나의 화면**에서 조회할 수 있게 한다.
- 사용자가 국민연금의 **매수/매도 동향을 시계열로** 파악할 수 있게 한다.

---

## 사용자 스토리

- **포트폴리오 조회 사용자**로서, 국민연금이 최근에 어떤 종목을 사고팔았는지 빠르게 확인하고 싶다.
- **포트폴리오 조회 사용자**로서, 특정 종목에서 국민연금의 지분율이 어떻게 변해왔는지 시계열로 보고 싶다.
- **포트폴리오 조회 사용자**로서, DART 공시 기준으로 가장 최근 변동이 있었던 종목 목록을 보고 싶다.
- **관리자**로서, DART 데이터를 수동으로 또는 자동으로 수집하여 DB를 최신 상태로 유지하고 싶다.

---

## 기능 요구사항

### 필수 (Must Have)

- [ ] 국민연금이 제출한 대량보유보고서를 DART에서 조회할 수 있다.
- [ ] 수집된 보고서에서 종목명, 보유 주식 수, 보유 비율, 공시 기준일을 추출할 수 있다.
- [ ] 추출한 데이터를 기존 보유 현황 조회 화면에서 함께 확인할 수 있다.
- [ ] DART 데이터와 공공데이터포털 데이터가 같은 종목에 대해 둘 다 존재할 경우, 출처를 구분하여 표시한다.
- [ ] 동일 종목에 공공데이터포털 데이터가 존재하면 해당 데이터를 우선 표시하고, DART 데이터는 보조 정보로 표시한다.
- [ ] 5% 미만 보유 종목의 DART 데이터는 별도 표기하여 일반 보유 종목과 구분한다.
- [ ] 관리자가 DART 수집을 수동으로 실행할 수 있다.
- [ ] DART 수집이 매일 1회 자동 실행된다.

### 선택 (Nice to Have)

- [ ] 새로운 공시가 등록됐을 때 사용자에게 알림을 제공한다.
- [ ] 최근 변동이 있었던 종목을 별도 목록으로 확인할 수 있다.

---

## 비기능 요구사항

- **데이터 정합성**: 동일 보고서가 중복 수집되지 않아야 한다.
- **응답 속도**: 기존 목록 및 상세 조회 응답 속도가 DART 데이터 추가 후에도 유지되어야 한다.
- **외부 의존성 격리**: DART API 장애 시 기존 공공데이터포털 기반 서비스에 영향을 주지 않아야 한다.
- **API 사용 한도 준수**: DART OpenAPI의 일일 요청 한도를 초과하지 않도록 수집 횟수를 제한한다.
- **설정 관리**: DART API 키는 관리자가 발급하며, 로컬·스테이징·운영 전 환경에 동일하게 적용된다.

---

## 범위 외

- DART 분기보고서·사업보고서 파싱은 이번 범위에 포함하지 않는다.
- 국민연금 외 다른 기관의 DART 공시 수집은 포함하지 않는다.
- 미국 주식 대상 SEC 13F 공시 수집은 포함하지 않는다.
- 새 공시 등록 시 실시간 알림(push/email) 기능은 포함하지 않는다.
- DART API 키 발급 절차 자동화는 포함하지 않는다.

---

## 성공 지표

- DART 수집 실행 후, 국민연금이 5% 이상 보유한 종목의 최근 지분 변동 이력을 서비스에서 확인할 수 있다.
- 동일 보고서를 여러 번 수집해도 데이터가 중복되지 않는다.
- 기존 공공데이터포털 데이터 조회 기능이 정상 작동한다 (회귀 없음).
- DART 수집 실패 시 기존 서비스에 영향이 없다.

---

## 결정 사항 (확정)

- **데이터 우선순위**: 동일 종목에 공공데이터포털 데이터가 존재하면 공공데이터포털 우선. DART 데이터는 보조 표기.
- **자동 수집 주기**: 일 1회.
- **API 키 관리**: 관리자가 직접 발급, 전 환경(로컬·스테이징·운영) 적용.
- **5% 미만 종목**: 수집 후 별도 표기하여 일반 보유 종목과 구분.
